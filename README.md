# XML 批次轉換工具 (xml-batch-convert)

把 CSV 對應表的每一列資料,套用到一個**範例 XML** 上,批次產生多個 XML 檔。輸出檔名由 [config.json](config.json) 的「匯出檔名規則」決定。

對應欄位、特殊規則、檔名格式都在 [config.json](config.json),可在 GUI 內直接編輯,**改完即自動存檔**,不用手動按儲存。

---

## 執行方式

**透過 MyTools Launcher**(建議):在 launcher 工具清單安裝後直接開啟。

**獨立執行**:需要 Python 3.8+(內建 tkinter,Windows 版預設就有)。

```powershell
python converttool.py
```

首次執行會自動建立 `config.json`(**空白骨架** — 無欄位對應、無特殊規則,
檔名格式留空)。獨立執行時放程式所在資料夾;透過 launcher 執行時放使用者
家目錄(`.xml_batch_convert_config.json`)。需要可用範例時,到「範例下載」
頁籤下載 `config.json` 範例再匯入。

## 介面結構

四個頁籤:

### 頁籤 1:CSV 產生 XML

從上到下:

1. **設定區** — 三組「輸入框 + 瀏覽」
   - CSV 對應表
   - 範例 XML
   - 輸出資料夾
2. **開始轉換** 按鈕
3. **進度條**
4. **結果摘要** 一行(轉換結束時顯示「成功 N、失敗 M、略過 K」)
5. **CSV 預覽 / 執行結果** 表格

選好 CSV 後**立刻**讀進來填到下方表格(不用等到按開始轉換)。表格欄位 = `#` + CSV 各欄(沿用 CSV 表頭) + `狀態`。轉換時會在「狀態」欄即時顯示每筆結果並上色:

| 顏色 | 狀態 | 例 |
|---|---|---|
| 綠底 | ✓ 成功 | `✓ 95181001.xml` |
| 紅底 | ✗ 失敗 | `✗ 檔名重複 [...]`、`✗ 失敗: ...` |
| 黃底 | 略過 | `略過: 欄位數 N < M`、`略過: 套用檔名格式後為空` |

### 頁籤 2:設定檔

從上到下:

1. **設定檔資訊**
   - 路徑(顯示目前載入的 .json 路徑)
   - **瀏覽** — 切換到別的設定檔
   - **另存新檔** — 把目前設定存成新檔並切過去
   - 狀態列(✓ 已載入 / ✓ 已自動儲存 / ⚠ 驗證失敗 / ✗ 寫入失敗)
2. **重新載入** / **下載預設設定**
3. **內嵌的設定編輯器**(三個子頁籤,以下說明)

### 頁籤 3:範例下載

提供三個內嵌範例下載(內容直接打包在程式內,不依賴任何外部檔):

| 範例 | 內容 |
|---|---|
| **下載範例 CSV** | `CSV對應表範例.csv` — 含表頭與兩筆示範資料 |
| **下載範例 XML** | `範例xml.xml` — 套用對應規則的基礎 XML 模板 |
| **下載範例設定** | `config.json` — 完整欄位對應與特殊規則,可直接匯入參考 |

每個範例一列(標題 + 說明 + 下載按鈕),按下後選擇儲存位置即可另存。
這裡的「下載範例設定」是**完整可用範例**(與下方〈設定檔範例〉相同),
跟「設定檔」頁籤的「下載預設設定」(空白骨架)不同。

### 頁籤 4:說明

工具使用說明(本文件的精簡版)。第 1 節「匯出檔名規則」也同步顯示在
「設定檔 → 匯出檔名規則」子頁籤,方便邊設定邊看。

## 設定編輯器

### 子頁籤:匯出檔名規則

輸入框「匯出檔案檔名:」,下方並顯示規則說明與範例(與本節相同內容)。
**預設為空白**,需自行填寫或從「範例下載」匯入範例設定。語法:

- `{A}` `{B}` `{C}` … 表示對應 CSV 欄位的值(支援字母 A-Z 或數字 0-based:`{0}` 等同 `{A}`)
- 其餘文字原樣保留
- 副檔名 `.xml` 自動加上,**不要寫**

範例:

| 規則 | 結果(假設 C=`C欄值`、A=`A欄值`) |
|---|---|
| `{C}` | C欄值.xml |
| `XML_{C}` | XML_C欄值.xml |
| `{C}_{A}` | C欄值_A欄值.xml |
| `114年_{C}_v1` | 114年_C欄值_v1.xml |

**空白檔名格式**:設定可留空且能正常載入(不報錯),但**按「開始轉換」時**
會擋下並提示先填寫,不會誤產生檔案。

**重複偵測**:兩筆以上資料算出同一檔名時,**所有重複的列**都不會輸出(避免互相覆蓋),預覽列顯示 `✗ 檔名重複`。

### 子頁籤:欄位對應

把 CSV 某欄的值寫到範例 XML 的指定行。表格欄位 + 右側按鈕(新增 / 編輯 / 刪除 / ↑↓):

| 欄位 | 說明 | 例 |
|---|---|---|
| 欄位 (column) | CSV 第幾欄,字母 A/B/C 或數字 0/1/2 | `C` |
| 行號 (line) | 範例 XML 第幾行(1-based) | `366` |
| 標籤 (tag) | XML 標籤名(不含 `<>`) | `gco:CharacterString` |
| 備註 (note) | 純註解,不影響執行 | `C 欄 (圖號)` |

**套用方式**(整行覆蓋,沿用原本縮排):

```
{原本縮排}<{tag}>{該欄值}</{tag}>
```

**注意**:
- 標籤只是輸出文字,**不會檢查範本原本的標籤是否一致**
- 範本若被重新格式化(改縮排/合併行),所有 line 都得對應更新
- 行號 < 1 或 > 範本總行數時會略過該規則並警告

### 子頁籤:特殊規則

目前只支援一種類型:**欄位為空時刪除指定行**

| 欄位 | 說明 | 例 |
|---|---|---|
| 名稱 | 純標示用,不影響執行 | `E 欄為空時刪除行 374~376` |
| 類型 | 下拉選單(目前一種) | 欄位為空時刪除指定行 |
| 欄位 (column) | 要檢查的 CSV 欄位 | `E` |
| 行號 (lines) | 多個用逗號分隔 | `374,375,376` |

**行為**:該欄為空字串時,把指定行**整列**從輸出 XML 中刪除(不只清空)。欄位有值時規則不動作。

**多條規則互不干擾**:條件判斷都用範本「原始行號」,刪除集中在最後一次處理,不會因前面刪了行而導致後面行號錯位。

## 處理順序

每筆 CSV 列依序進行:

1. 從**乾淨的範本** XML 開始(每筆都從頭來)
2. 套用所有「欄位對應」(mappings)
3. 套用所有「特殊規則」(special_rules) — 收集要刪的行,一次刪掉
4. 套用「匯出檔名規則」算出檔名,寫成 `.xml`

**注意**:mappings 寫到的內容若該行剛好被特殊規則刪除,內容也會一併消失。一般情況這就是你要的(該欄為空,所以連帶把對應段落整段砍掉)。

## 自動儲存

設定編輯器**沒有「儲存」按鈕** — 任何更改即時寫回 `config.json`。

| 操作 | 寫檔時機 |
|---|---|
| 改「匯出檔案檔名」 | 停手 0.5 秒後寫(防抖,避免邊打邊存) |
| 表格新增 / 編輯 | 立即 |
| 表格刪除 / 上下移 | 立即 |

**狀態回饋**(設定檔 tab 上方狀態列,不彈窗):
- **✓ 已自動儲存** — 綠字
- **⚠ 驗證失敗 (未儲存)** — 紅字(例如檔名格式只打了 `{`,還沒打完整)
- **✗ 寫入失敗** — 紅字(檔案被佔用之類)

**「重新載入」** 是把硬碟上的 config.json 讀回畫面,可以在外部修改檔案後同步,或丟掉中途無效的編輯內容。

**「下載預設設定」** 把內建預設值(**空白骨架**)另存成新 `.json`,**不會覆蓋目前設定**。
要完整可用的參考範本,請到「範例下載」頁籤的「下載範例設定」。

## CSV 格式要求

- 第一列**必須是 header**(會被跳過,只用來在預覽表格顯示欄位名)
- 編碼支援 **UTF-8(含/不含 BOM)** 與 **CP950**,自動偵測
- 任何欄位空白都當作空字串處理
- 同一欄位可被多個 mapping 引用(例如 C 欄同時當作檔名和寫入第 366 行)

## 設定檔範例

**預設(首次啟動 / 下載預設設定)是空白骨架**:

```json
{
  "filename_template": "",
  "min_required_columns": 1,
  "mappings": [],
  "special_rules": []
}
```

下方為**完整範例**(「範例下載」→「下載範例設定」會輸出此內容):

```json
{
  "filename_template": "{C}",
  "min_required_columns": 1,
  "mappings": [
    {"column": "A", "line": 5,   "tag": "gco:CharacterString", "note": "A 欄"},
    {"column": "B", "line": 181, "tag": "gco:CharacterString", "note": "B 欄"},
    {"column": "C", "line": 366, "tag": "gco:CharacterString", "note": "C 欄 (圖號)"},
    {"column": "D", "line": 372, "tag": "gco:CharacterString", "note": "D 欄"},
    {"column": "E", "line": 375, "tag": "gco:CharacterString", "note": "E 欄"},
    {"column": "F", "line": 448, "tag": "gco:Decimal",         "note": "F 欄"},
    {"column": "G", "line": 451, "tag": "gco:Decimal",         "note": "G 欄"},
    {"column": "H", "line": 454, "tag": "gco:Decimal",         "note": "H 欄"},
    {"column": "I", "line": 457, "tag": "gco:Decimal",         "note": "I 欄"}
  ],
  "special_rules": [
    {
      "name": "E 欄為空時刪除行 374~376",
      "type": "clear_lines_if_column_empty",
      "column": "E",
      "lines": [374, 375, 376]
    }
  ]
}
```

| 頂層欄位 | 型別 | 說明 |
|---|---|---|
| `filename_template` | string | 必填,匯出檔名規則 |
| `min_required_columns` | int | 每列最少欄位數,不足會被略過(預設 1) |
| `mappings` | 陣列 | 欄位對應,每條物件含 `column` / `line` / `tag` / `note` |
| `special_rules` | 陣列 | 特殊規則,每條物件含 `name` / `type` / `column` / `lines` |

## 自訂範例

### 想新增「J 欄 → 第 500 行」

開「設定檔」頁籤 → 「欄位對應」子頁籤 → 點「新增」:
- 欄位:`J`
- 行號:`500`
- 標籤:`gco:CharacterString`

按確定後**立即寫入** `config.json`,不用按存檔。

### 想換成 D 欄當作檔名,並加前綴

「匯出檔名規則」子頁籤 → 把輸入框改成 `XML_{D}` → 等 0.5 秒,自動存檔。

### 想新增「F 欄為空時刪除行 448」

「特殊規則」子頁籤 → 點「新增」:
- 名稱:`F 欄為空時刪除 448`
- 類型:欄位為空時刪除指定行
- 欄位:`F`
- 行號:`448`

確定後立即生效。

## 訊息符號

| 符號 | 意義 |
|---|---|
| ✓ | 成功產生 |
| ✗ | 失敗或檔名重複,未輸出 |
| 略過 | 該列被跳過(欄位數不足、套用檔名格式後為空等) |
| ⚠ | 警告(例如範例 XML 行數不足某條 mapping/rule 的行號) |

## 常見問題

**Q: 提示「檔名重複」怎麼辦?**
A: 兩筆以上資料的「匯出檔名規則」算出來的檔名相同。檢查 CSV 是否真的有重複的圖號,或調整檔名規則加入更多欄位(例如 `{C}_{D}`)使其唯一。

**Q: 提示「範例 XML 行數不足」?**
A: 範例 XML 的總行數小於設定中最大的 line 值。檢查模板是否被裁切,或調整 mappings/rules 的行號。

**Q: 改了 config.json 但 GUI 沒反應?**
A: 點設定檔頁籤的「重新載入」即可同步。

**Q: 輸出的 XML 縮排跟範例不一致?**
A: 縮排會直接沿用範例 XML 該行原本的空白。檢查範例該行開頭的空格/tab 是否符合預期。

**Q: 想要一份可用的範例設定?**
A: 到「範例下載」頁籤點「下載範例設定」,存成 `.json` 後用「設定檔」頁籤的
「瀏覽」切換過去即可。「下載預設設定」給的是空白骨架,不是完整範例。

**Q: 想把設定清空重來?**
A: 點「下載預設設定」存到目前 `config.json` 路徑(會詢問確認覆蓋),即得空白骨架。

**Q: config.json 在哪?**
A: 獨立執行時在程式所在資料夾;透過 MyTools Launcher 執行時在使用者家目錄
(`.xml_batch_convert_config.json`)。
